- 난이도: ★☆☆
- 예상 작업일: 1일
- 권장 주차: Week 1 1일, Week 2 0일
- 핵심 기능: 연락처/텍스트/이메일/Wi-Fi 기반 QR 생성과 스타일 커스터마이징, 저장 및 공유 파이프라인을 가진 생성 유틸 앱.

## 2) 2주 실행 계획
### Week 1

- [ ] W1-$((2 + 1)): P1 결과 라우팅과 히스토리 정합성 점검
### Week 2
- [ ] W2-1: Week 2는 폴리싱/회귀 확인 및 문서 정리만 수행
- [ ] W2-2: 운영 반영 체크리스트 반영(릴리즈 전 점검 항목 정리)

## 3) 현재 버그 가능성 추정
| 위험 영역 | 발생 확률 | 영향도 | 점검 포인트 |
| --- | --- | --- | --- |
| 입력 형식 파싱 오류 | 높음 | 높음 | 연락처/URL/JSON/질문 세트 파싱 실패 시 예외 처리 확인. |
| 카메라/권한 처리 미흡(스캐너) | 중 | 높음 | 권한 거부 상태에서 스캔 플로우의 fallback 경로 확인. |
| 데이터 공유 실패 | 중 | 중 | 저장/공유 채널 미지원 시 사용자 안내 문구와 재시도 정책 검증. |
| 세션 데이터 외부 호출 실패 영향 | 중 | 중 | 네트워크 지연 시 타임아웃/재요청 처리 점검. |
| 권한 변경 후 UI 갱신 누락 | 중 | 낮음 | 권한 상태 변경 event listener 동기화 확인. |

## 4) 회귀 테스트 케이스
- [ ] TC-IO-01: 입력 텍스트(연락처/URL/Wi-Fi)가 특수문자 포함 상태로도 파싱되어 출력되는지 확인
- [ ] TC-IO-02: 카메라 권한 거부/회수 상태에서 스캐너가 적절한 안내를 노출하고 중단되지 않는지 확인
- [ ] TC-IO-03: 생성/공유 결과가 빈 값이 아닌 포맷으로 앱 외부 공유되는지 확인
- [ ] TC-IO-04: 오프라인/저속 네트워크 환경에서 캐시된 데이터로 동일 동작이 유지되는지 확인
- [ ] TC-IO-05: 라우팅 이동 중 잘못된 payload가 들어왔을 때 예외 처리되어 홈으로 복귀하는지 확인
